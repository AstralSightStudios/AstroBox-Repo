所有喜爱、关注与支持 AstroBox 的用户大家好！感谢你参与 AstroBox 2.0 的公测！

在上个版本发布后，有许多小伙伴加入了我们的公开测试。我们收到了非常多反馈，感谢所有小伙伴填写的问卷！在 2.0.1 中，我们围绕资源安装、UI/UX细节、网络可用性、账号稳定性和桌面端连接体验做了集中改进，同时微调了移动端性能体验、插件市场和发布构建相关问题。

## 新增功能
  - 新增 GitHub DoH 加速选项，可只针对 GitHub 资源使用 DoH 解析，改善部分网络环境下资源源、插件市场、图片和下载链接加载失败的问题
  - 新增账号注销流程，支持注销前检查与工单提交
  - 新增评论通知支持，信箱现在可以处理评论类消息（可能不完善），通知展示和阅读体验也做了拆分优化
  - 表盘安装新增强制安装能力，用于处理表盘 ID 冲突场景
  - 插件市场安装改为接入全局下载队列，支持更完整的下载进度展示
  - 新增 OOBE 许可步骤，并调整首次启动导航流程
  - 新增 Microsoft Clarity 数据收集隐私开关

## 资源与设备安装
  - 小米表盘 ID 支持 9 位或 12 位字母数字组合，不再只按纯数字处理
  - 新增小米可穿戴固件识别逻辑
  - 安装固件前增加文件校验，减少误选文件或损坏文件导致的安装风险
  - 下载按钮现在展示资源版本与更明确的详情标签
  - 资源详情页布局继续优化，引入更平滑的胶囊式信息组件
  - 首页合集支持一键加入队列，资源预览图尺寸和瀑布流加载体验也有改进

## 网络与资源源
  - 官方资源源加载健壮性提升，刷新失败时会进入失败状态，不再长时间停留在更新中
  - 新增多个 GitHub 公共代理镜像，并统一前后端 GitHub CDN 逻辑
  - 默认网络客户端增加连接与读取超时，避免请求长期挂起
  - 启动时账号节点探测改为后端探测，修复大陆/国际账号源自动切换误判
  - 插件市场、固件下载和资源图片在 CDN/DoH 场景下的处理逻辑更一致

## 账号与会员
  - AstroBox OAuth 增加 refresh token 支持，登录态过期后会优先尝试刷新
  - 账号刷新失败、鉴权错误和会话过期时的处理更稳健，并会向前端发出账号更新事件
  - 个人主页会员展示、机型判断和账号资料展示做了优化

## 界面体验
  - 首页探索流新增作者链接和骨架加载
  - 优化首页搜索浮层中的全局搜索
  - 优化移动端导航设备卡片电量样式
  - 路由动画接入用户的减少动画偏好
  - 设备页更新状态改为更直观的状态指示
  - 修复米坛源资源简介表格内容显示问题
  - 未登录状态下隐藏评分弹层，并禁用部分需要账号的交互
  - 字体资源重新整理，修复部分字体加载和显示问题

## 插件与桌面端
  - Windows 经典蓝牙 SPP 不再每次连接都强制配对
  - 修复 Windows SPP 传输过程中可能卡死的问题
  - 修复 Windows 停止搜索蓝牙设备时界面被阻塞的问题
  - 插件列表状态现在会在多个 Tauri 窗口之间同步
  - 插件系统文件对话框、保存文件会话和下载目录安装流程做了稳定性调整

## 平台与构建
  - 尽可能优化了移动端的流畅度
  - 为 iOS App Store 发布做了一些调整
  - 升级 Tauri 相关依赖
